## I. Popper 对 RASHOMON 的三个根本贡献

### 贡献一：证伪主义 = RASHOMON 的认识论立场

RASHOMON 不声称"找到真理"——它声称"排除错误"。这正是 Popper 的证伪主义。

**精确映射**：
| Popper | RASHOMON |
|--------|----------|
| 猜想 (conjecture) | R1 代理的声明 |
| 反驳 (refutation) | R2 对抗性审查 |
| 暂时接受 (tentative acceptance) | Partial Order Consensus |
| 证伪 (falsification) | ≥2/3 refuters 驳斥 → 丢弃 |

**RASHOMON 的核心 claim 需要可证伪条件**：
- "在 N 场辩论中，如果 RASHOMON 的结论与单代理分析的结论无统计显著差异，则方法无效"
- "在 M 个案例中，如果 RASHOMON 遗漏了单代理能发现的错误，则方法有缺陷"

### 贡献二：渐进社会工程 = RASHOMON 的发展策略

Popper 区分整体社会工程（从头设计完美社会）和渐进社会工程（逐步解决具体问题）。RASHOMON 应采用渐进策略：
- 不追求"终极版本"
- 每次改进都是可证伪的——如果改进无效，可以回滚
- 从具体案例中学习，不依赖抽象理论

### 贡献三：开放社会 = RASHOMON 的政治目标

RASHOMON 应服务于开放社会——增强而非替代公民判断。如果 RASHOMON 变成了"AI 独裁者"（用户完全依赖系统判断），它就违背了开放社会的原则。

## II. Popper 批判柏拉图对 RASHOMON 的启示

Popper 对柏拉图的批判虽然被经典学者认为过度简化，但对 RASHOMON 有直接警示：

1. **哲学王 → AI 独裁**：如果 RASHOMON 的裁决不被质疑，它就成了"AI 哲学王"
2. **高贵谎言 → 分层披露**：RASHOMON 不应欺骗用户，但可以简化复杂性
3. **诗歌审查 → 代理审查**：如果 RASHOMON 过度约束代理的推理自由，它就在做柏拉图式的审查

## III. Kuhn 和 Lakatos 的修正

Kuhn (1962) 论证：科学家不是每天都在证伪——他们通常在"正常科学"（paradigm 内工作）中积累知识。只在危机时刻才发生"范式转换"。

**RASHOMON 的映射**：RASHOMON 的大多数辩论是"正常科学"——在既定框架内工作。只有在 YNI > 0.85 时才发生"范式转换"——系统承认当前框架可能有根本性问题。

Lakatos (1970) 修正了证伪主义：科学理论有"硬核"（不可证伪的核心假设）和"保护带"（可被修正的辅助假设）。

**RASHOMON 的映射**：
- 硬核 = "多视角审查优于单视角"（RASHOMON 的核心假设）
- 保护带 = 具体的阈值、代理数量、审查轮数（可被修正）
